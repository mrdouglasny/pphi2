# pphi2 assurance — local settings

This project follows the conventions of
[`math-commons/formalization-assurance`](https://github.com/math-commons/formalization-assurance)
— verification (kernel + axiom certificate), assumption review (per-axiom vetting),
faithfulness (informal↔formal correspondence), and validation (acceptance ladder).

This file declares pphi2's **local settings**; it does not duplicate the generic
content. For the model and naming conventions see the hub's `README.md`,
`VERIFICATION_VALIDATION.md`, `ADOPTION.md`, `AXIOM_AUDIT_FORMAT.md`, `VETTING.md`,
and `FORMALIZATION_YAML.md`.

## Mode

**Mixed forward/backward chaining.** The project is mostly **backward-chaining** —
a posed target spec (P(Φ)₂ as a Wightman QFT in 1+1d via the OS axioms), with
remaining analytic content staged as `axiom`s pending discharge. The forward
component is the OS axiom system itself, formalized as `Pphi2.SatisfiesFullOS`
(matching Osterwalder–Schrader 1973).

**Assurance spine:** the backward chain dominates, so the spine is
`AXIOM_AUDIT.md` (top-level, narrative history) + `audit/vetting/` (per-axiom
soundness records) + the comparator path (when the headline theorem is
sorry/axiom-clean).

## CI gates (live)

The hub's shared workflow ([`assure.yml`](https://github.com/math-commons/formalization-assurance/blob/main/.github/workflows/assure.yml))
is wired via [`.github/workflows/assurance.yml`](../.github/workflows/assurance.yml)
(one-line caller of the hub workflow). Reads `audit/vetting/policy.yml`
for strictness.

At the current **L1** strictness the gates run on every push/PR and:
- **warn** if `audit/axiom-report.txt` is out of sync with the kernel trace
  (regenerated via `lake env lean audit/axiom_report.lean` and `diff -u`);
- **warn** if a `sorry` appears outside `audit/sorry-allowlist.txt`
  (scans `git ls-files '*.lean'` with comment-stripping);
- vetting-coverage enforcement (every axiom has a record in `audit/vetting/`)
  is not yet in the reusable workflow — currently a manual gate at L2/L3.

Both checks emit `::error::` annotations and step-summary lines at L1 but
exit 0, so the check status stays green. Raising to **L2** in
`policy.yml` flips them to enforced (failures block).

Coverage is already 19/19 (see [`vetting/README.md`](vetting/README.md)),
so L2 is reachable as soon as the per-axiom metadata stabilizes.
